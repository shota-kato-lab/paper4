Universal β Predictor

Predict urban scaling exponent β from network structure alone — zero parameters, zero labels (Path A) — Kato Framework Paper 4 companion tool

What is the scaling exponent β and how is it predicted? ▼

β is a number describing how a city-level indicator (e.g., GDP, patent filings, total road length, gasoline consumption) scales with city population at a single point in time. If a city has 10× the population of another, the indicator is approximately 10β times larger (e.g., β = 1.1 ⇒ about 12.6× larger). This is not a year-over-year growth rate, but a cross-sectional comparison between cities of different sizes.

On this site, we measure β from network structure alone, with zero parameters and zero labels (Path A), using a method developed by the author.

Notation: βpred = network-based prediction (does not use βobs). βobs = observation value computed from per-city indicator data (N, Y).

Foundational theory: Scaling Exponent Hierarchy Law (author's original framework)

β = 1 ± 1 / [H · ln(2H + 1)],   H ∈ ℕ

H ∈ {1, …, 8} = relay-depth steps. The number of intermediate hops that people, information, and money traverse when coordinating within a city. H = 1 corresponds to direct, simple coordination; H = 8 represents a deeply stacked hierarchical structure.

The integer H and the sign (±) are mathematically determined uniquely by the type of network (city, biological, quantum, etc.) and the physical constraints of the substrate. No coefficient fitting is performed against observed data.

A single relay-depth law connecting integer hierarchy depth H and scaling exponent β. The constraint H ∈ ℕ expresses that hierarchy depth is driven by human population N.

This site offers two analysis paths — Path A (auto-classify, true blind) and Path B (labeled, with structural hints). When both are valid, the user may choose which path to use (default = Path A).

How do these paths predict β from a network? ▼

Path A — Auto-classify (true blind, zero label). Given only an edgelist (no panel-level metadata), the detector extracts graph features (degree distribution, average clustering, k-core, Louvain modularity) and runs a 4-step pipeline to recover (network_type, substrate, branch_sign, H_max) → β_pred = β±(H).

The detector skips Step 1's auto-classification and uses the labels to seed Steps 2-4. Important: these are panel-level structural labels, not per-edge or per-node tags.

    How do these methods predict β from a network? ▼
    

      
Path A — Auto-classify (true blind, zero label). Given only an edgelist (no panel-level metadata), the detector extracts graph features (degree distribution, average clustering, k-core, Louvain modularity) and runs a 4-step pipeline:

      
① Step 0 cluster count. Louvain modularity Q + module-size distribution + boundary blur + size CV → noise_score ∈ [0, 1]. If Q < 0.2 → single cluster; Q ≥ 0.3 → multi-cluster (Tier B mixture territory).

      
② Step 1 H_max classification. From clustering & k-core, infer network_type ∈ {phys_infra, socio_econ, hybrid}, substrate ∈ {area_bounded, volume_bounded, unbounded_dens, mixed}, and integer H_max ∈ {1,…,8}.

      
③ Step 2 adjacent-pair assumption. H_pair = {H_max − 1, H_max} (= "hierarchy jump not allowed").

      
④ Step 3 replicator dynamics. For mixture rows, solve for mixture weights π_low, π_up using cluster sizes & structural priors.

      
⑤ Step 4 dual β output. β_pred_clean = β±(dominant H) (= pure-network prediction). β_pred_contaminated = β±(H_eff) where H_eff = π_low · H_low + π_up · H_up (= mixture-aware prediction). The bracketing interval [β_clean, β_contaminated] reports the noise-direction span.

      
Path B — Labeled (with structural hints). User supplies (or the Bettencourt panel sample provides) the structural metadata directly: network_type, substrate, branch_sign (+/−/balanced), H_max ∈ {1,…,8}, is_mixture (boolean). The detector skips Step 1's auto-classification and uses the labels to seed Steps 2-4. Important: these are panel-level structural labels (one set per indicator row, e.g., "this row is a socio-econ unbounded-density row at H=2, +-branch"), not per-edge or per-node tags.

      
Important note on data semantics. The Bettencourt 22-panel sample buttons supply a synthetic edgelist (configured to mirror the structural type each indicator implies, per V261 §S-PanelOrigins) plus the panel-level labels. The synthetic graph is built only from network type / substrate / branch / H_max — it never reads β_obs. Bettencourt's original 2007 data are the per-city (N, Y) pairs, summarized only as β_obs ± CI in this tool's UI; they are not used to compute β_pred.

      
What does "zero label" actually mean? In Path A, zero label = zero structural hint (no H, no network_type, no branch sign, not even a name) — the detector recovers everything from the bare adjacency list. Path B trades that blindness for one extra structural hint set, giving a more direct β_pred for that panel row but losing the "blind discovery" claim.

About

  
This tool implements the Universal β predictor from the Kato Framework (Paper 4: A Relay-Depth Exponent Ladder Closing the Bettencourt 22-Indicator Urban-Scaling Panel).

  
The framework reaches a 22/22 closure on the Bettencourt 2007 22-indicator panel (synthetic by-construction; real data verification in progress).

  
References: paper PDF / Zenodo deposit / GitHub source. Author: Shota Kato (AI & Future Co · Independent research).
